HTTP 0.9

Simple, one-line protocol. Only returned HTML files using the GET method.

HTTP 1.0

  • Added request and response headers
  • Supported different content types
  • Opened a new TCP connection for each request

HTTP 1.1

  • Persistent connections by default
  • Improved caching
  • Host header required to support many sites on one server
  • Supported pipelining to reduce delays

HTTP 2

  • Introduced binary framing
  • Multiplexing supports many streams on one connection
  • Header compression reduces overhead
  • Greatly improves speed on modern websites

HTTP 3

  • Uses QUIC instead of TCP
  • Faster recovery from packet loss
  • Built for mobile networks and unstable connections
  • Becoming widely adopted by major platforms
